HTML

  • The head element is a container for metadata about the webpage. Information in the head element is not visible to end users.
  • The body element represents visible content shown to the user.
  • The meta elements contain data about page and are used by the browser.
  • Changing the title element within the head will only change the text dispalyed in the browser's tab.
  • Section elements define sections in document
  • Best practice is to use semantic elements when possible
  • Ordered lists (ol) are numbered.
  • Unordered lists (ul) are bullet points.

Cascading Style Sheets (CSS)

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • text-align: right, left, center
  • color can be specified by name or hex #
  • font-size: #px

Git

  1. Edit code
  2. Use git status to verify the current branch.
  3. Stage all changes (git add -A).
  4. Commit changes (git commit -m "message").
  5. Pull from main branch to make sure all files are up to date (git pull origin main)
  6. Push the changes to the current feature branch (git push origin branch).
  7. Go to GitHub to create a pull request
  8. Merge the changes to update the main branch after having them reviewed by a colleague.
  9. Update main branch using GitBash (git checkout main; git pull origin main).

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git checkout branch name: switches to the desired branch
  • git add -A: add command adds modifactions in current working branch to the staging area, and the -A switch adds all changes
  • git commit -m "message": commits the changes
  • git pull origin branch: pulls source code to ensure all code is up to date and avoids conflicts
  • git push origin branch: pushes changes
  • code .: opens VS Code from the CLI
  • touch "name of file": creates a file in the local directory

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Remember to use a semicolon after each line of code.
  • Make sure variables are defined before trying to call them in our code (see Control Flow).
  • Unlike our CSS file, the JavaScript is linked in the body element of the HTML file.
